# PKA Integration Plans

Status: Draft  
Owner: Product + Backend + UI + Agent Protocol

## Summary

This folder contains planning documents for integrating selected PKA-inspired ideas into Paperclip.

Paperclip remains the base platform. These plans are intended to improve operator clarity, delegation quality, and durable memory without weakening Paperclip's existing control plane, approval model, activity logging, or company boundaries.

## Scope Rules

- Adopt only ideas that add clear product or operator value.
- Keep Paperclip's issue, approval, activity, budget, and adapter architecture intact.
- Prefer structured entities and APIs over filesystem protocols or markdown-as-database conventions.
- Treat PKA as a source of workflow and information-architecture ideas, not as a system to merge directly.
